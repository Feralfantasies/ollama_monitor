## Home Assistant Custom Integration for ollama_monitor

Create a Home Assistant custom integration (`custom_components/ollama_monitor/`) that non-technical users can install and configure **entirely through the HA GUI** — no YAML, no terminal, no MQTT dependency.

### What gets built

**Directory structure** (inside the ollama_monitor repo):
```
ha_integration/custom_components/ollama_monitor/
├── manifest.json
├── __init__.py
├── config_flow.py
├── sensor.py
├── coordinator.py
├── const.py
└── strings.json
```

**GUI Setup Flow:** User navigates *Settings → Devices & Services → Add Integration*, searches "Ollama Monitor", enters the monitor URL (default `http://192.168.1.1:3000`), and confirms. The config flow validates connectivity by hitting `/api/status` before saving.

**Sensors created automatically** (all properly typed for HA statistics/graphing):
- `sensor.ollama_monitor_ollama_status` — available/unreachable + loaded model, model count
- `sensor.ollama_monitor_gpu_temperature` — °C with temperature state class
- `sensor.ollama_monitor_gpu_memory_used` — MiB with data state class
- `sensor.ollama_monitor_gpu_utilization` — % with percentage state class
- `sensor.ollama_monitor_gpu_power` — W with power state class

Uses `DataUpdateCoordinator` so all sensors share one `/api/status` poll per cycle (matching the monitor's refresh interval).

**README updated** with two installation methods (HACS preferred, manual copy fallback), sensor reference table, and troubleshooting.
